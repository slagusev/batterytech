BOOL32 BatteryTech::Context::isUIConsumingKeys

Set to true if the UI is consuming the current input keys. Your app can use this to decide if it wants to ignore keys the UI is responding to. This is used instead of an event chain of consumption pattern.

BOOL32 BatteryTech::Context::isUIConsumingPointers

Set to true if the UI is consuming the current pointers. Your app can use this to decide if it wants to ignore touches the UI is responding to. This is used instead of an event chain of consumption pattern.

U8 BatteryTech::Context::keyPressed

For keyboard input, this is the last key the user pressed and is cleared by the UI library once consumed. As of Batterytech 1.0 - key events are not queued so if the user types faster than the FPS of the app, keys will be missed.

KeyState* BatteryTech::Context::keyState

For keyboard input, this is an array of the keys that are currently pressed down. This array will be of MAX_KEYSTATES length.

MenuRenderer* BatteryTech::Context::menuRenderer

BOOL32 BatteryTech::Context::newGraphicsContext

Set to true if the graphics context was lost and there is a new graphics context. If this is set to true, you must reload all VRAM objects (textures, FBOs, VBOs, etc) and reset the GL states before continuing.

PointerState* BatteryTech::Context::pointerState
