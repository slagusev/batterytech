File Asset Reading and Writing

File Assets, File IO, Directories and Paths. More...

Detailed Description

Reading may happen from an Android APK, iOS/OSX Bundle or a plain old Windows, Linux or other file system. Many of the supported input systems can not be written to so it is recommended that an application developer always plan on writing to external or application storage (whichever is more appropriate) instead of ever assuming that the input files may ever be written to.

Loading and freeing an asset example:

S32 size;
unsigned char* data = _platform_load_asset("myasset.dat", &size);
if (data) {
// process data of size
}
_platform_free_asset(data);
